Custom MCP Server

Built for technicians first, developers second Slide 4
GitLab Integration Branch and Hook Workflow

Branch-Driven Development and Auto Refresh

Step 1
Engineer commits updates in GitLab branch (iex_cpu source or rules).
Step 2
post-commit hook triggers build_index.py in background.
Step 3
Vector DB refreshes fault, parameter, and source contexts.
Step 4
Agent uses updated MCP tools on next request without manual sync.
Step 5
Team validates behavior with fault and parameter regression prompts.
This workflow keeps answers aligned with latest branch changes Slide 5
